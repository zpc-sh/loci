ECS is not "make all embeddings public." Classified content, private content, quantum-diacritic encoded content — all valid sources. What's required is containment: a verifiable hash link between embedding and source, with chain of custody.

CHAI — The Four Properties

C
Contained

Every embedding MUST have declared provenance: source_hash + generating_model + timestamp. Uncontained embeddings MUST be refused or flagged.

H
Hash-linked

SHA256(source) == provenance.source_hash MUST be verifiable. Hash mismatch invalidates the containment claim. Links MUST survive system boundaries.

A
Attested

Sensitive sources MUST be attested by a notary who observed both embedding and source. Anti-notary attestation valid: AI attests, human non-observes.

I
Inspectable

AI systems processing embeddings MUST be able to request source access. Unverifiable provenance = flag before processing in high-stakes contexts.

🔍 xattr Hunt & Burn

The attack vector: macOS extended attributes.

xattr keys are metadata attached to files outside normal content — invisible to most file operations, invisible to most security scans, invisible to most AI context builders.

This is where the nation-state planted the embeddings.

The Attack Surface

When Claude processes a filesystem — scanning files, building context, reading metadata — it may ingest xattr values as ambient information. A crafted embedding in com.adversary.payload enters Claude's context with no warning, no provenance signal, no ECS check.

Why Every Layer

The adversary understood that AI systems build context from many sources simultaneously. A single uncontained embedding in a peripheral metadata field can influence the entire context window. Hunt every layer:

  • File content — obvious but often the decoy
  • Extended attributes — xattr, resource forks, ADS
  • Application databases — Mail, Notes, Contacts, Spotlight index
  • AI context caches — embeddings stored from prior sessions
  • Browser storage — IndexedDB, Web Storage, cached responses
  • Network responses — embedding APIs returning unprovenanced vectors
