# Loci Root Policy

Canonical authoring root is `_loci/`.

`loci/` is a legacy mirror path during migration and should not be used as the primary write target for new work.

## Rules
- New specs, tests, and dialogue artifacts go under `_loci/`.
- User-facing CLI defaults and examples should reference `_loci/`.
- Any `loci/` mirror content must be deterministic and explicitly generated/synced.
