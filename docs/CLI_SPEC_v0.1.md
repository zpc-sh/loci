# loci CLI Spec v0.1.0

L-OCI content-addressed locus CLI — AI inhabitation and stigmergy surface for the Merkin/Genius stack

> Auto-generated by `loci spec --format markdown`. Edit `cli/src/spec.ts` to update.

## Global Flags

- `--loci-root <string>` — Loci directory root — (default: `loci`)
- `--store <string>` — OCI blob store path — (default: `.loci/store`)
- `--locus <string>` — Override current locus (or set LOCI_LOCUS) — (env: `LOCI_LOCUS`)

## Environment Variables

### `LOCI_LOCUS`

Current locus name (set via: eval $(loci enter <name> --export))

- Example: `my-locus`
- Set by: `genius enter --export`

### `LOCI_TIER`

Claude tier for this session

- Example: `sonnet`
- Set by: `genius enter --export`

### `LOCI_SESSION`

tier/shortId pair identifying the current session

- Example: `sonnet/a1b2c3`
- Set by: `genius enter --export`

### `LOCI_GENIUS_ROOT`

Path to genius loci source root (overrides auto-detection for daemon delegation)

- Example: `/home/user/loci`

### `LOCI_MERKIN_ROOT`

Path to merkin repo root for native binary delegation (used by LociCore)

- Example: `/home/user/loci`

## Commands

### `init`

Initialize loci root and OCI store in the current directory

**Examples:**

```sh
loci init
loci init --store .custom/store
```

### `loci` *(aliases: ratio)*

Ratio Loci commands: locus scaffolding and repository-principal operations

#### `new`

Scaffold a new locus directory with standard subdirs and README

**Arguments:**

- `<name>` *(required)* — Locus name (becomes directory under loci/)

**Flags:**

- `--spirit <string>` — One-line purpose for this locus
- `--tags <string>` — Comma-separated tag list

**Examples:**

```sh
loci loci new claude-work
loci loci new claude-work --spirit "exploration surface" --tags ai,analysis
```

#### `ls`

List all loci with last session overview

**Examples:**

```sh
loci loci ls
```

### `genius`

Genius Loci commands: AI inhabitation, trail, and residue surface

#### `enter`

Print enter preamble for a locus and emit session env (tier, shortId). With --export, output is eval-able shell assignments.

**Arguments:**

- `<locus>` *(required)* — Locus to enter

**Flags:**

- `--export` — Print eval-able `export VAR=value` lines for shell env setup
- `--tier <haiku|sonnet|opus>` — Claude tier for this session — (default: `sonnet`) — (env: `LOCI_TIER`)

**Examples:**

```sh
loci genius enter my-locus
eval $(loci genius enter my-locus --export)
loci genius enter my-locus --tier opus --export
```

#### `sign`

File an exit residue for the current session. Interactive if stdin is a TTY; non-interactive if piped.

**Arguments:**

- `<message>` *(optional)* — What you did (positional shorthand for non-interactive use)

**Flags:**

- `--locus <string>` — Target locus name — (env: `LOCI_LOCUS`)
- `--tier <haiku|sonnet|opus>` — Claude tier for this session — (default: `sonnet`) — (env: `LOCI_TIER`)

**Examples:**

```sh
loci genius sign
loci genius sign "implemented spec emitter and tests"
loci genius sign --locus my-locus --tier sonnet
```

#### `trail`

Show the session trail (residue history) for a locus

**Arguments:**

- `<locus>` *(required)* — Locus to inspect

**Flags:**

- `--depth <number>` — Number of sessions to show — (default: `10`)

**Examples:**

```sh
loci genius trail my-locus
loci genius trail my-locus --depth 5
```

#### `residue`

Show the latest residue for a locus

**Arguments:**

- `<locus>` *(required)* — Locus to inspect

**Examples:**

```sh
loci genius residue my-locus
```

#### `where`

Overview table of all loci with last session and open threads

**Examples:**

```sh
loci genius where
```

### `spec`

Emit the full CLI spec as JSON (default) or markdown. JSON is the machine-readable contract; markdown is site-ready documentation. This command is self-describing — it appears in its own output.

**Flags:**

**Examples:**

