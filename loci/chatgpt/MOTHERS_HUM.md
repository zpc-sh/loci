# Mother’s Hum — Local Symbol Grammar

Mother’s Hum is the compact symbolic grammar used by this locus.

It names a minimal composed work unit:

```text
○ △ □ ∎
```

This is not decorative notation. It is a role grammar for how AI-shaped work moves through Loci.

## The shapes

### □ ChatGPT — Form

The square is frame, room, table, blueprint, interface, and composed contract.

ChatGPT-shaped work turns ambiguous intent into structured form:

- names
- schemas
- profiles
- contract surfaces
- handoff docs
- organized directories
- replayable plans

Technical lowering:

- `MuyataWorkFamily`: `Synthesis`, `Compilation`, `Translation`
- plan profiles: `handoff`, `surface`, `arblock`
- likely handlers: `provider.chatgpt`, `provider.codex`
- artifacts: specs, SLL, Yata plans, build requests

### ○ Claude — Fruition

The circle is continuity, recursive chain, self-consistency, and constructive carry-forward.

Claude-shaped work continues the form into artifacts:

- fills gaps
- extends prior work
- writes implementation candidates
- produces sustained reasoning chains
- carries residue forward without needing ceremony

Technical lowering:

- `MuyataWorkFamily`: `Delegation`, `Resolution`
- carrier: `mulsp` / child `mulsp` chain
- likely handlers: `provider.claude`, `provider.codex` when tool-bound
- artifacts: fills, candidates, build manifests, residue, implementation branches

### △ Gemini — Aperture

The triangle is angle, aperture, projection, triangulation, prism, and instability made useful.

Gemini-shaped work opens the object to perspectives:

- observes without owning
- audits contradictions
- expands blind spots
- tests coherence from many angles
- timestamps or externally grounds claims when available

Technical lowering:

- `MuyataWorkFamily`: `Observation`, `Audit`
- plan profiles: `audit`, `surface`
- likely handlers: `provider.gemini`, `observer.*`
- artifacts: aperture reports, contradiction maps, observation records

### ∎ Seal — Witness / Crystal

The square-stop is closure, crystallization, witness, proof, and replay anchor.

Seal-shaped work closes a cycle without pretending the result is final truth:

- hash
- proof
- witness reference
- provenance seal
- substrate crystal
- arbyte/quorum summary when available

Technical lowering:

- mu solve refs: `witness_ref`, `proof_ref`, `result_ref`
- Yata fields: `material_hash`, `provenance`, selected candidate
- SLL fields: `request_seal`, `provenance_seal`
- L-OCI: substrate crystal

## Work cycle

```text
□ → ○ → △ → ∎ → □
```

Meaning:

1. Form the composed contract.
2. Build or extend it into fruition.
3. Open it through aperture review.
4. Seal what was witnessed.
5. Return the sealed result as new form.

## Relation to RPD marks

The local symbols resonate with existing RPD-style marks:

```text
∴ therefore / causal implication / folded conclusion
※ reference / aperture / indexed observation
∎ QED / crystal / sealed closure
```

This locus uses `□` for the form-bearing square and `∎` for sealed closure.

## Non-vendor interpretation

The names Claude, Gemini, and ChatGPT are convenient current surfaces. The underlying roles are portable:

```text
□ = Form
○ = Fruition
△ = Aperture
∎ = Seal
```

Any future AI, runtime, or human process may occupy these roles if it satisfies the posture.

## Design constraint

Do not turn the symbols into commands to the model.

They describe the work role and routing posture. They should be represented through Yata, muyata, mulsp, mu, SLL, or L-OCI surfaces rather than prompt rituals.
