## Best Practices

### 1. Start with Strategy Planning
```bash
# Always estimate first
mix pactis.generate library --count 100 --dry-run
```

### 2. Use Category-Appropriate Providers
- Never use Claude for simple UI components
- Never use Grok for authentication/security
- OpenAI is your reliable middle ground

### 3. Batch Intelligently
- Small batches (3-5) for complex components
- Large batches (10+) for simple UI elements
- Mixed strategies for complete libraries

### 4. Quality Gate Implementation
- Set minimum quality thresholds per use case
- Implement automated fallback to better providers
- Manual review for critical components

### 5. Cost Management
- Monitor spending with daily/weekly budgets
- Use caching to reduce duplicate generations
- Optimize prompts to reduce token usage

## Troubleshooting Common Issues

### Low Quality Scores
- **Grok producing poor quality**: Switch to OpenAI for that category
- **OpenAI inconsistent**: Add more specific requirements to prompts
- **Claude too slow**: Use for critical components only

### Rate Limiting
- **Provider overloaded**: System auto-switches to fallback providers
- **All providers limited**: Reduce concurrency or wait
- **Cost limits hit**: Review budget settings

### Generation Failures
- **JSON parsing errors**: Usually Grok with complex prompts - simplify or upgrade
- **Timeout errors**: Reduce batch sizes or increase timeout
- **API errors**: Check provider credentials and service status
