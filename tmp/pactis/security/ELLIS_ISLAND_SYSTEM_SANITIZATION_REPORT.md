Assessment:
- No obvious oversized blob set detected.
- The `.so` is a legitimate candidate for deeper trust verification (checksum provenance/signing), since native artifacts can execute arbitrary machine code when loaded.

## Recommended next hardening step
- Add a CI (or local policy script) that blocks commits containing:
  - new native binaries (`*.so`, `*.dylib`, `*.dll`, `*.exe`) unless explicitly allowlisted,
  - large artifacts above a threshold (for example 1MB or 5MB),
  - unexpected executable bits in non-script paths.
