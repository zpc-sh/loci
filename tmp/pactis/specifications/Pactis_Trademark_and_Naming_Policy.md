# Pactis™ Trademark & Naming Policy

Status: draft — not legal advice

## Scope
This policy governs use of the Pactis™ name and related identifiers in documentation, software packages, services, and marketing.

## Ownership and Purpose
- Pactis™ identifies an open specification stewarded by Pactis.
- The mark distinguishes authentic references and compatible implementations, reducing user confusion.

## Permitted Uses (no permission required)
- Descriptive references: “implements the Pactis specification”, “compatible with the Pactis‑TVI interface”.
- Non‑commercial documentation: tutorials, research, reviews with proper attribution.
- Package names with clear namespace: `cdfm_pactis_*`, `@pactis/pactis-*`, `pactis.pactis.*`.

## Uses Requiring Permission
- Product or service branding primarily identified as “Pactis”.
- Visual marks/logos implying official status.
- Conformance badges using the word “Certified”.

## Prohibited Uses
- Misleading claims of affiliation or stewardship.
- Using “Pactis” as the dominant brand for unrelated products.
- Look‑alike logos or marks.

## Conformance Badges
- Allowed without prior approval when criteria are met:
  - “Pactis‑Compatible” — passes published interface tests for a Pactis profile (e.g., TVI, API).
  - “Pactis‑Ready” — integrates Pactis context policy and determinism guarantees; not a certification.
- Reserved (permission required):
  - “Pactis‑Certified” — reserved for an official conformance program.

## Namespacing Guidance
- Packages: `cdfm_pactis_*` (Hex), `@pactis/pactis-*` (npm), `pactis-pactis-*` (crates), modules `Pactis.Pactis.*`.
- Avoid bare `pactis` as a top‑level package unless owned by Pactis.

## Non‑Affiliation Disclaimer
Use this statement in first prominent references: “Pactis™ is an open specification by Pactis; not affiliated with any other entity using similar names. ™ indicates an unregistered trademark claim.”

## Contact and Permissions
For permissions or questions, contact the Pactis project maintainers.
