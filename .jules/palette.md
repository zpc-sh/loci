## 2024-05-24 - `@clack/prompts` Cancellation Fallthrough Prevention
**Learning:** When using `@clack/prompts` for sequential terminal prompts, pressing Ctrl+C returns a symbol checked by `clack.isCancel()`. However, if individual prompts in a multi-prompt sequence don't each check for this condition and call `process.exit(0)`, the sequence will incorrectly "fall through", potentially filling subsequent variables with default values or undefined states instead of actually terminating the CLI application. Furthermore, calling `clack.cancel()` without a string message causes strict typing errors in some contexts, degrading developer experience.
**Action:** Always provide a string message to `clack.cancel()` (e.g., `clack.cancel('Cancelled.')`) for better UX and type safety. More importantly, in multi-prompt sequences, ensure `clack.isCancel()` is checked for *every* individual prompt followed by an explicit `process.exit(0)` to properly abort the workflow on user cancellation.
