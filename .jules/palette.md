## 2024-05-24 - Handling multi-prompt CLI cancellations
**Learning:** When using @clack/prompts in a sequence, a single cancellation via Ctrl+C doesn't automatically halt the entire flow if the script blindly continues evaluating subsequent prompts. If `clack.isCancel()` is not checked individually for every prompt step, the user experience breaks: the sequence will ignore the cancellation request, fall through incorrectly, and still record incomplete or corrupt data. Furthermore, always providing a string argument to `clack.cancel(message)` prevents runtime errors from strict typing when executing in strict environments.
**Action:** Always check `if (clack.isCancel(result))` for *every* individual prompt response in a multi-step sequence, and specifically call `clack.cancel('Operation cancelled.'); process.exit(0)` to properly abort execution.
