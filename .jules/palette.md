## 2026-05-03 - Consistent Cancellation for clack.text Prompts\n**Learning:** When using @clack/prompts for a sequence of inputs, failing to explicitly handle cancellation for every prompt (via `clack.isCancel()`) can lead to unintended progression or malformed residues if the user hits Ctrl+C on a subsequent prompt. Furthermore, `clack.cancel()` should always include a string message to provide clear visual feedback.\n**Action:** When adding or reviewing multi-prompt sequences, ensure every step checks `if (clack.isCancel(result)) { clack.cancel('Cancelled'); process.exit(0); }` before proceeding.
